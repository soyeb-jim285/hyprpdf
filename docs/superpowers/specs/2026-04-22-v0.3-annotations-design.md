# HyprPDF v0.3 Annotations — Design

**Date:** 2026-04-22
**Status:** Approved (brainstorm phase)
**Author:** Jim (soyeb-jim285) + Claude
**Scope:** v0.3 milestone per `OUTLINE.md` — full annotation set: highlight, underline, strikeout, sticky note, freehand ink; 6-color palette; in-session undo/redo; Markdown export.

## Goals

- Persistent per-document annotation store keyed by SHA-256 of file content so rename/move keeps notes intact.
- Five annotation types covering the 80/20 of annotating workflow: highlight, underline, strikeout, sticky note, freehand ink.
- Hybrid entry UX: text annots via popup on existing text selection; sticky + ink via toolbar tool toggles.
- In-session undo/redo (depth 50, per-tab).
- Markdown export grouped by page.
- Reuses v0.2 text-selection pipeline (char-merged word rects from Poppler) — no new selection infra.

## Non-Goals

- Bake annotations into the PDF file (Poppler write-path) — deferred to v1.0.
- Rectangle, ellipse, arrow, line, typewriter text-box primitives — deferred to v1.1.
- Comment threads / reply chains on annotations — deferred to v1.1.
- Dedicated annotations side-panel browser — deferred (search panel space already budgeted).
- Pen pressure / pressure-variable stroke width — deferred.
- Bezier smoothing on ink strokes — deferred.

## Architecture

Three layers, additive to v0.2:

- **Backend (C++):** `DocumentHasher` computes SHA-256 off-thread with mtime cache. `PdfDoc` exposes `contentHash` Q_PROPERTY. `AnnotationStore` is a `QAbstractListModel` holding the current doc's annotations, loaded/saved as JSON sidecar keyed by hash, with in-session undo/redo stack.
- **QML overlay:** `AnnotLayer.qml` inside each `PdfView` page delegate renders all five annot types via type-dispatched delegates. Reuses v0.2 under-text shader for highlights. `SelectionPopup.qml` floats near drag-release offering 3 text-annot buttons + 6 color swatches. `AnnotToolbar.qml` is a slim row for sticky + ink modal tools. `StickyEditor.qml` and `AnnotContextPopup.qml` handle per-annot editing.
- **System:** JSON sidecars at `~/.local/share/hyprpdf/annotations/<sha256>.json`. `QCryptographicHash::Sha256`, `QtConcurrent::run` for hashing, `QSaveFile` for atomic writes.

## Backend

### JSON schema (`~/.local/share/hyprpdf/annotations/<sha256>.json`)

```json
{
  "version": 1,
  "docHash": "<sha256-hex>",
  "annotations": [
    {
      "id": "<uuid>",
      "type": "highlight|underline|strikeout|stickyNote|ink",
      "page": 0,
      "rects":      [[x,y,w,h], ...],   // highlight/underline/strikeout only
      "anchor":     [x, y],              // stickyNote only
      "strokes":    [[[x,y],...], ...],  // ink only
      "strokeWidth": 2.5,                // ink only
      "color": "#RRGGBB",
      "author": "Jim",
      "createdAt": "<ISO-8601>",
      "note": ""                         // stickyNote body; empty for other types
    }
  ]
}
```

All coordinates in **PDF points** — same space as `PdfDoc::selectionRectsLine`.

### `DocumentHasher`

```cpp
class DocumentHasher : public QObject {
    Q_OBJECT
public:
    QFuture<QString> hashFile(const QString &path);     // async, 8 MiB chunks
    QString hashFileSync(const QString &path);          // sync fallback
private:
    QCache<QString, QString> m_cache;                   // key: abs path#mtime#size
};
```

Runs on `QThreadPool::globalInstance()`. Returns lowercase hex. Cache keeps last 64 hashes.

### `PdfDoc` additions

```cpp
Q_PROPERTY(QString contentHash READ contentHash NOTIFY contentHashReady)

// On load(), enqueue DocumentHasher->hashFile(path). Emit contentHashReady when
// future finishes. Synchronous getter blocks only if called before the async
// completes (rare — QML bindings wait for NOTIFY).
QString contentHash() const;
signals:
    void contentHashReady(const QString &hash);
```

### `AnnotationStore`

```cpp
class AnnotationStore : public QAbstractListModel {
    Q_OBJECT
    Q_PROPERTY(int count          READ rowCount       NOTIFY countChanged)
    Q_PROPERTY(int revision       READ revision       NOTIFY revisionChanged)
    Q_PROPERTY(QString selectedId READ selectedId     WRITE setSelectedId NOTIFY selectedIdChanged)
    Q_PROPERTY(bool canUndo       READ canUndo        NOTIFY historyChanged)
    Q_PROPERTY(bool canRedo       READ canRedo        NOTIFY historyChanged)
    Q_PROPERTY(QString currentAuthor READ currentAuthor WRITE setCurrentAuthor NOTIFY currentAuthorChanged)

public:
    enum Type { Highlight = 0, Underline, Strikeout, StickyNote, Ink };
    Q_ENUM(Type)

    enum Roles {
        IdRole = Qt::UserRole + 1,
        TypeRole, PageRole, RectsRole, AnchorRole,
        InkStrokesRole, InkStrokeWidthRole,
        ColorRole, NoteRole, AuthorRole, CreatedAtRole, SelectedRole
    };

    Q_INVOKABLE void loadDocument(const QString &hash);   // clears + loads sidecar
    Q_INVOKABLE void clear();

    Q_INVOKABLE QString addHighlight(int page, QVariantList rects, const QColor &color);
    Q_INVOKABLE QString addUnderline(int page, QVariantList rects, const QColor &color);
    Q_INVOKABLE QString addStrikeout(int page, QVariantList rects, const QColor &color);
    Q_INVOKABLE QString addStickyNote(int page, QPointF anchor, const QColor &color,
                                      const QString &note);
    Q_INVOKABLE QString addInk(int page, QVariantList strokes, const QColor &color,
                               qreal strokeWidth);

    Q_INVOKABLE void remove(const QString &id);
    Q_INVOKABLE void setColor(const QString &id, const QColor &color);
    Q_INVOKABLE void setNote(const QString &id, const QString &note);

    Q_INVOKABLE QVariantList annotationsOnPage(int page) const;

    Q_INVOKABLE void undo();
    Q_INVOKABLE void redo();

    Q_INVOKABLE void exportMarkdown(const QString &path) const;
};
```

Undo/redo internally uses `QStack<Command>`; each add/remove/setColor/setNote pushes an inverse-Command. Depth capped at 50 — oldest entry evicted on overflow. `revisionChanged` monotonically increments on every mutation so QML Repeater bindings re-evaluate.

Save is debounced (300 ms `QTimer`) after any mutation; atomic write via `QSaveFile`.

### Load/save flow

1. `documentModel.currentIndexChanged` → `annotationStore.loadDocument(pdfDoc.contentHash)`.
2. Store reads `~/.local/share/hyprpdf/annotations/<hash>.json` if exists; else starts empty.
3. Mutations schedule save 300 ms later; save writes same path atomically.

### Threading

- GUI thread: store mutations, QML updates.
- Worker (QtConcurrent): SHA-256 hashing, JSON read/write.
- Poppler: main thread (unchanged from v0.2).

## Text annotation pipeline (reuses v0.2 selection)

1. User drag-selects on page. v0.2 MouseArea fills `_selectionByPage[pageIdx]` with char-merged word rects in PDF pts.
2. On `onReleased` with non-empty selection, `PdfView` emits `textSelected(int page, QVariantList rects, QPointF scenePt)`.
3. `Main.qml` shows `SelectionPopup` anchored near `scenePt`: 3 type buttons (Highlight / Underline / Strikeout) + 6 color swatches.
4. Click type + color → `annotationStore.addHighlight(page, rects, color)` (etc.). Popup closes. Selection cleared.
5. Click outside popup → dismisses without creating annot. Selection stays until next click.

## Modal tools (sticky + ink)

### `AnnotToolbar.qml`

Collapsible slim row between Toolbar and TabsBar. Toggle via `F8` shortcut or main-Toolbar "annotate" button.

Contents (left to right):
- Sticky tool toggle (reuses existing `IconPin` from quill-icons)
- Ink tool toggle (`IconPen`)
- Eraser / None button (`IconEraser`) — cancels active tool, returns to selection mode
- Divider
- Highlight tool swatch, Underline swatch, Strikeout swatch (for users who prefer tool-first over selection-first; sets active-text-tool that applies on next selection)
- 6 color swatches (yellow/green/pink/blue/orange/purple) — sets `activeAnnotColor`
- Slider (1–6 px) visible only when Ink tool active — sets `activeInkWidth`

Active tool stored as `ApplicationWindow.activeAnnotTool` enum.

### Sticky tool

- Cursor becomes `Qt.CrossCursor` over page.
- Single click on page → `addStickyNote(page, clickPt, activeColor, "")` → opens `StickyEditor` for body.
- Tool stays active until Esc.

### Ink tool

- `onPressed`: start new stroke, capture points at 2 px deltas.
- `onPositionChanged`: push px to current-stroke array, render live preview via `Shape { ShapePath { PathPolyline } }`.
- `onReleased`: convert px→pts, `addInk(page, [stroke], color, width)`, clear buffer.

### Esc exits any tool back to None.

## Rendering (`AnnotLayer.qml`)

Per-page overlay inside `PdfView` page delegate, above page Image, below search highlights.

Repeater model reactive on `annotationStore.revision`. Delegate `Loader` dispatches by type:

- **Highlight** — per-rect Item + `ShaderEffectSource` + `ShaderEffect` using v0.2 `highlight_under_text.frag.qsb`. `tintColor` = annot color. `opacity: 0.55`.
- **Underline** — Rectangle per rect, height 1.5 px, anchored to bottom of rect. `color: annot.color`.
- **Strikeout** — Rectangle per rect, height 1.5 px, anchored vertical center.
- **StickyNote** — pin icon (24 px) at anchor, `opacity: 0.9`, `color: annot.color`. Click → `StickyEditor` for edit. Selected state: 2 px accent border.
- **Ink** — `Shape { ShapePath { PathPolyline {path: stroke_points} } }` per stroke. `strokeWidth: annot.strokeWidth`, `strokeColor: annot.color`, `fillColor: transparent`, `RoundCap`/`RoundJoin`.

### Click-to-select

Each delegate has a transparent MouseArea that emits `annotationClicked(id, type, clickScenePt)`. Main.qml routes to `AnnotContextPopup`. Store's `selectedId` updates; visual selection = 2 px accent border on the annot.

## `AnnotContextPopup.qml`

Shown on annotation click. Anchored near click point.

Contents:
- 6 color swatches (click → `annotationStore.setColor(id, color)`)
- "Edit note" button (sticky only → opens `StickyEditor`)
- "Delete" button (→ `annotationStore.remove(id)`)

Closes on Esc, click-outside, or after action.

`Del` key shortcut at `ApplicationWindow` level: if `selectedId` set, call `remove(selectedId)`.

## Undo / redo

- `Ctrl+Z` → `annotationStore.undo()`
- `Ctrl+Y` / `Ctrl+Shift+Z` → `annotationStore.redo()`
- Depth 50 ring buffer, per-doc (cleared on tab switch via `loadDocument`).
- Each mutation pushes an inverse-Command onto the undo stack and clears the redo stack. Undo pops from undo, applies inverse, pushes to redo.
- Mutations suppress save debounce during undo/redo until the history op completes.

## Markdown export

`annotationStore.exportMarkdown(path)`:

1. Sort annots by (page, createdAt).
2. Write header: file name, SHA-256, export timestamp, author.
3. Group by page with `## Page N` headers.
4. Per annot:
   - Highlight / Underline / Strikeout: `- **Type** \`#color\` — "<textInRect of rects bounding box, trimmed>"`
   - StickyNote: `- **Note** \`#color\` at (X, Y) — <note body>`
   - Ink: `- **Ink** \`#color\` (N strokes)`
5. Atomic write via `QSaveFile`.

Example output shown in §5 of brainstorm (see git log for full sample).

Triggered from File menu entry ("Export annotations...") or shortcut (`Ctrl+Shift+E`).

## Error handling

- Sidecar JSON parse fail → rename to `<hash>.json.broken-<unix-ts>`, start empty. Log.
- Sidecar write fail → `qWarning` + retry once; if still fails, surface to QML via `saveFailed` signal.
- Hash failure (unreadable file, permission, corrupt read) → return empty string; `AnnotationStore.loadDocument("")` → no-op, in-memory only.
- Broken annotation record (missing required fields) → skip + log, don't abort load of rest.
- Undo on empty stack → no-op.
- Remove unknown id → no-op.
