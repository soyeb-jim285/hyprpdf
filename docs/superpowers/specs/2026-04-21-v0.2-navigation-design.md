# HyprPDF v0.2 Navigation — Design

**Date:** 2026-04-21
**Status:** Approved (brainstorm phase)
**Author:** Jim (soyeb-jim285) + Claude
**Scope:** v0.2 milestone per `OUTLINE.md` — all seven navigation features: thumbnails sidebar, outline/TOC panel, text search, text selection + copy, expanded keyboard shortcuts, dark/invert colors mode, collapsible panels.

## Goals

- Ship a full Okular-style PDF navigation experience built on the v0.1 Poppler-based renderer.
- Add per-tab search with visible match highlights and navigation (F3/Shift+F3).
- Enable text selection via drag-rect with clipboard copy (lays foundation for v0.3 highlight annotations reusing the same rect→text pipeline).
- Expose document outline as a tree panel synced with the current page.
- Ship a dark-invert shader toggle for low-light reading.

## Non-Goals

- Word-level caret text selection (deferred to v0.3 or later).
- Async thumbnail render queue (YAGNI — reuse existing `PdfPageImageProvider`, upgrade only if stutter is measured).
- Async search (QtConcurrent) — synchronous for v0.2; upgrade path documented.
- Settings UI panel — belongs to v0.6 polish milestone.
- Form filling, digital signatures, forms (v2+).

## Architecture

Three layers, additive to v0.1:

- **Backend (C++):** `PdfDoc` gains `search(page, text)`, `textInRect(page, rect)`, `toc()`; new `SearchController` (per-app QObject driving match state); new `OutlineModel : QAbstractItemModel` (tree of Poppler outline); new `ClipboardBridge` (QObject wrap around `QGuiApplication::clipboard()`).
- **QML panels:** `ThumbnailList.qml` (left), `OutlinePanel.qml` (right), `SearchBar.qml` (Ctrl+F overlay). `PdfView.qml` gains text-select MouseArea, search-match overlay Repeater, dark-invert ShaderEffect, `goToPage(idx)` + `scrollToMatch(page, rect)` API. `Main.qml` becomes `SplitView` horizontal with collapsible side panels and new Action/Shortcut block.
- **System:** No new runtime deps. Wayland clipboard via `QClipboard` (Qt handles wl-copy/xclip transparently).

## Backend

### `PdfDoc` additions

```cpp
// new Q_INVOKABLE accessors; existing API unchanged
Q_INVOKABLE QVariantList search(int page, const QString &text) const;
// returns list of QRectF (PDF points). Empty if no matches or invalid page.

Q_INVOKABLE QString textInRect(int page, const QRectF &rect) const;
// returns concatenated text of words intersecting rect (PDF points).

struct TocEntry {
    QString title;
    int pageIndex;       // -1 for entries without a page destination
    QVector<TocEntry> children;
};
const QVector<TocEntry> &toc() const;   // built lazily on first call
```

Implementation:
- `search` iterates `Poppler::Page::search(text, caseSensitivity, diacriticsMode)` using `Poppler::Page::IgnoreCase | IgnoreDiacritics`.
- `textInRect` calls `Poppler::Page::text(rect)`; returns trimmed `QString`.
- `toc` walks `Poppler::Document::outline()` recursively. Each `OutlineItem::destination()` resolves via `Poppler::LinkDestination::pageNumber() - 1` (Poppler is 1-indexed, we are 0-indexed). Cached in `mutable` field.

### `SearchController`

```cpp
class SearchController : public QObject {
    Q_OBJECT
    Q_PROPERTY(QString query       READ query        NOTIFY queryChanged)
    Q_PROPERTY(int matchCount      READ matchCount   NOTIFY resultsChanged)
    Q_PROPERTY(int currentIndex    READ currentIndex NOTIFY currentChanged)
    Q_PROPERTY(int currentPage     READ currentPage  NOTIFY currentChanged)
    Q_PROPERTY(QRectF currentRect  READ currentRect  NOTIFY currentChanged)

public:
    struct Match { int page; QRectF rect; };

    Q_INVOKABLE void setDocument(QObject *doc);   // PdfDoc* cast; nullptr clears
    Q_INVOKABLE void search(const QString &q);
    Q_INVOKABLE void next();
    Q_INVOKABLE void prev();
    Q_INVOKABLE void clear();
    Q_INVOKABLE QVariantList matchesOnPage(int page) const;   // QRectF list for overlay

signals:
    void queryChanged();
    void resultsChanged();
    void currentChanged();
    void scrollToRequested(int page, QRectF rect);
};
```

Synchronous search on the main thread. On `search(q)`, iterates all pages, stores `QList<Match>`, sets `currentIndex` to 0 if results exist. `next/prev` wrap modulo matchCount and emit `scrollToRequested`. `setDocument(nullptr)` clears.

### `OutlineModel : QAbstractItemModel`

```cpp
class OutlineModel : public QAbstractItemModel {
    Q_OBJECT
public:
    enum Roles {
        TitleRole = Qt::DisplayRole,
        PageRole = Qt::UserRole + 1,
        DepthRole
    };

    Q_INVOKABLE void setDocument(QObject *doc);   // PdfDoc* or nullptr

    QModelIndex index(int row, int col, const QModelIndex &parent = {}) const override;
    QModelIndex parent(const QModelIndex &child) const override;
    int rowCount(const QModelIndex &parent = {}) const override;
    int columnCount(const QModelIndex &parent = {}) const override { return 1; }
    QVariant data(const QModelIndex &index, int role = Qt::DisplayRole) const override;
    QHash<int, QByteArray> roleNames() const override;

private:
    struct Node {
        QString title;
        int page = -1;
        Node *parent = nullptr;
        QVector<Node*> children;
    };
    Node *m_root = nullptr;
    void buildFromToc(const QVector<PdfDoc::TocEntry> &src, Node *parent);
};
```

Bound in QML via `TreeView` (Qt 6.4+). `setDocument(nullptr)` resets to empty root.

### `ClipboardBridge`

```cpp
class ClipboardBridge : public QObject {
    Q_OBJECT
public:
    Q_INVOKABLE void setText(const QString &s);
    Q_INVOKABLE QString text() const;
};
```

Wraps `QGuiApplication::clipboard()`. Registered as `clipboard` context property in `main.cpp`.

### Threading

All synchronous on main thread. Poppler is not thread-safe per-document. Search <500ms for typical docs — acceptable. Upgrade to `QtConcurrent::run` only if stutter observed.

## QML

### `ThumbnailList.qml`

Left panel. `ListView` of delegates: white page rect + `Image` source `image://pdf/<id>/<idx>/<width>` + 1-indexed page number label below. Highlight current page with accent border. Click → emits `pageActivated(page)` which `Main.qml` routes to `curView().goToPage(page)`. `cache: true` on the `Image` (thumbs stay warm). Width default 180 px (from SplitView preferred).

### `OutlinePanel.qml`

Right panel. `TreeView { model: outlineModel }` with delegate showing indented title. Empty state: centered "No outline" label. Click → `pageActivated(pageRole)`. Width default 240 px.

### `SearchBar.qml`

Overlay, anchored top-center of the PdfView region. `TextField` + match counter + prev/next chevrons + close X. Live-search on text edit (debounced 150 ms via `Timer`). Return triggers `controller.next()`. Escape closes and clears. Exposes `openWithFocus()`.

### `PdfView.qml` additions

Per-page delegate gains:

1. **Text-select MouseArea** — press/drag/release captures rubberband rect in pixels; on release converts to PDF points via `pxPerPt = targetW / pageSize.width`; calls `document.textInRect(index, rectPts)`; if non-empty, `clipboard.setText(text)`.
2. **Search-match overlay** — `Repeater { model: searchController.matchesOnPage(index) }` draws translucent `Theme.warning` rectangles at converted positions. Current match gets 2-px border.
3. **Dark-invert ShaderEffect** — parallel to Image, `visible: invertColors`. Fragment shader `vec4(1.0 - c.rgb, c.a)`. Falls back to `QQuickPaintedItem` with `CompositionMode_Difference` if the inline string shader is rejected by RHI; detection at runtime — log + fall back.
4. **Current page tracking** — `ListView.onContentYChanged` recomputes `currentPage` from `indexAt(0, contentY + height*0.3)`. Bound by Main.qml to sync thumbnail + outline highlight.

Public API additions:
- `goToPage(idx)` — `positionViewAtIndex(idx, ListView.Beginning)`.
- `scrollToMatch(page, rectPts)` — scrolls to page.
- `setInvertColors(b)` — sets the property.

### `Main.qml` layout

`ColumnLayout { Toolbar; TabsBar; SplitView { ThumbnailList; [Stack+SearchBar]; OutlinePanel } }`. Panel visibility driven by `root.leftPanelVisible` / `root.rightPanelVisible` booleans. `Connections` on `documentModel.currentIndexChanged` rebinds `searchController.setDocument` and `outlineModel.setDocument` to the current tab's `PdfDoc`.

## Error handling

- Search with empty string → clear results silently.
- `textInRect` on rect that spans no words → returns empty string (no clipboard write).
- Dark-invert shader compile failure → log `qWarning`, fall back to unshaded Image (invert button becomes no-op with log, doesn't crash).
- Outline on doc without TOC → empty model, panel shows "No outline" placeholder.
- Panel toggle during resize → SplitView.visible transitions handled by Qt internally.
